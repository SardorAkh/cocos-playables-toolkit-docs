ScenarioSystem

Async step runner for Cocos Creator 3.8.x Playable Ads Async step runner для Cocos Creator 3.8.x Playable Ads 面向 Cocos Creator 3.8.x 试玩广告的异步步骤运行器

ScenarioSystem is a linear async/await scenario runner for managing sequential game flows. Lifecycle-safe, abort-aware, ~1.5KB gzipped. ScenarioSystem — линейный async/await runner сценариев для управления последовательными игровыми потоками. Lifecycle-safe, с поддержкой abort, ~1.5KB gzip. ScenarioSystem 是一个线性 async/await 场景运行器，用于管理顺序游戏流程。生命周期安全，支持中止，gzip 后约 1.5KB。

💡 Design philosophyПринцип проектирования设计理念

No timeouts by default. In playable ads, the player decides when to act — the system should never skip a step because of inactivity. Set timeoutMs explicitly only where needed. Нет таймаутов по умолчанию. В playable ads игрок сам решает когда действовать — система не должна пропускать шаг из-за бездействия. Ставь timeoutMs явно только где нужно. 默认无超时。在试玩广告中，玩家决定何时操作——系统不应因不活动而跳过步骤。仅在需要时显式设置 timeoutMs。

Quick Start #

import { ScenarioRunner, createScenario, WaitStep, CallbackStep, WaitForInputStep } from "./ScenarioSystem";

@ccclass("GamePlayable")
class GamePlayable extends Component {
    @property(Node) startBtn: Node = null!;
    private runner!: ScenarioRunner;

    protected start(): void {
        this.runner = this.addComponent(ScenarioRunner);

        createScenario()
            .wait(1)
            .waitForInput(this.startBtn)
            .call(() => console.log("Game started!"))
            .debug(true)
            .buildAndRun(this.runner);
    }

    protected onDestroy(): void {
        this.runner?.abort();
    }
}

💡 ParallelStep + failFast

When failFast = true, the first error aborts all sibling steps automatically. Without failFast, all steps run to completion and errors are collected. При failFast = true первая ошибка автоматически абортит все параллельные степы. Без failFast все степы выполняются до конца, ошибки собираются. 当 failFast = true 时，第一个错误会自动中止所有并行步骤。不使用 failFast 时，所有步骤运行到完成，错误会被收集。

Custom Steps #

Extend BaseStep. Use this.signal for abort, PromiseHelper for async. Расширяй BaseStep. Используй this.signal для abort, PromiseHelper для async. 扩展 BaseStep。使用 this.signal 进行中止，PromiseHelper 进行异步操作。 

class ShowRewardStep extends BaseStep {
    constructor(
        private rewardNode: Node,
        private coins: number,
        options: Partial<StepOptions> = {}
    ) {
        super({ name: "ShowReward", ...options });
    }

    protected async doExecute(context: StepContext): Promise<void> {
        this.throwIfAborted();
        this.rewardNode.active = true;

        const label = this.rewardNode.getComponent(Label);
        label.string = `+${this.coins} coins!`;

        await PromiseHelper.wait(context.component, 2, this.signal);
    }
}
⚠️ Rules for custom stepsПравила для кастомных степов自定义步骤规则

Use this.signal, not your own AbortController. Use PromiseHelper, not raw setTimeout/setInterval. Override abort() only to clean up external resources — always call super.abort(). Используй this.signal, а не свой AbortController. Используй PromiseHelper, а не голый setTimeout/setInterval. Переопределяй abort() только для очистки внешних ресурсов — всегда вызывай super.abort(). 使用 this.signal，而不是自己的 AbortController。使用 PromiseHelper，而不是原生 setTimeout/setInterval。仅在清理外部资源时覆盖 abort()——始终调用 super.abort()。

Step Factories #

Group related steps into factories. Captures shared dependencies, keeps scenarios clean. Группируй связанные степы в фабрики. Захватывает зависимости, сценарии остаются чистыми. 将相关步骤分组到工厂中。捕获共享依赖，保持场景简洁。 

// CameraSteps.ts
export function cameraSteps(service: CameraZoomService) {
    return {
        zoomTo(target: Node, zoomLevel = 1.5, duration = 0.5) {
            return new ZoomToTargetStep(service, target, zoomLevel, duration);
        },
        reset(duration = 0.5) {
            return new ResetCameraStep(service, duration);
        }
    };
}

Usage with DIИспользование с DI配合 DI 使用

@NeedInject()
@ccclass("MergePlayable")
class MergePlayable extends Component {
    @InjectProperty(CameraZoomService)
    private cameraZoom!: CameraZoomService;

    private runner!: ScenarioRunner;
    private cam!: ReturnType<typeof cameraSteps>;

    protected start(): void {
        this.runner = this.addComponent(ScenarioRunner);
        this.cam = cameraSteps(this.cameraZoom);

        createScenario()
            .addStep(this.cam.zoomTo(this.character, 2.0, 0))
            .wait(1.5)
            .addStep(this.cam.reset(0.8))
            .call(() => this.showCTA())
            .debug(true)
            .buildAndRun(this.runner);
    }
}
💡 Keep ScenarioSystem cleanДержи ScenarioSystem чистым保持 ScenarioSystem 纯净

Don't bundle domain steps (camera, merge, animation) into ScenarioSystem. Keep them as separate step-packs — each project picks what it needs. Не пакуй доменные степы (камера, мёрж, анимация) в ScenarioSystem. Держи их как отдельные step-packs — каждый проект берёт что нужно. 不要将领域步骤（相机、合并、动画）打包到 ScenarioSystem 中。将它们作为独立的 step-packs——每个项目选择所需的。
