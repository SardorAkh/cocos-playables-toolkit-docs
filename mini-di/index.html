MiniDI

Lightweight Dependency Injection for Cocos Creator 3.x Playable Ads Легковесный Dependency Injection для Cocos Creator 3.x Playable Ads 面向 Cocos Creator 3.x 试玩广告的轻量级依赖注入

MiniDI is a lightweight DI container inspired by VContainer (Unity). Optimized for playable ads: ~1.5KB gzip, no reflect-metadata. MiniDI — легковесный DI-контейнер, вдохновлённый VContainer (Unity). Оптимизирован для playable ads: ~1.5KB gzip, без reflect-metadata. MiniDI 是一个轻量级 DI 容器，灵感来自 VContainer (Unity)。专为试玩广告优化：~1.5KB gzip，无需 reflect-metadata。

💡 Key difference from VContainerКлючевое отличие от VContainer与 VContainer 的关键区别

In VContainer you can register MonoBehaviour in the container. In MiniDI, Cocos Components are NOT registered — they receive dependencies via @NeedInject() and @InjectProperty(). В VContainer можно регистрировать MonoBehaviour в контейнере. В MiniDI Cocos Components НЕ регистрируются, а инжектятся через @NeedInject() и @InjectProperty(). 在 VContainer 中可以将 MonoBehaviour 注册到容器。在 MiniDI 中，Cocos Components 不注册到容器，而是通过 @NeedInject() 和 @InjectProperty() 接收依赖。

Quick Start #

1. Create a serviceСоздай сервис创建服务

@Injectable([])
class AudioService {
    play(sound: string): void { }
}

2. Create LifetimeScopeСоздай LifetimeScope创建 LifetimeScope

@ccclass('GameLifetimeScope')
class GameLifetimeScope extends LifetimeScope {
    protected configure(builder: ContainerBuilder): void {
        builder.registerSingleton(AudioService);
    }
}

3. Inject into componentИнжекть в компонент注入到组件

@NeedInject()
@ccclass('PlayerController')
class PlayerController extends Component {
    @InjectProperty(AudioService)
    private audio!: AudioService;

    start(): void {
        this.audio.play('jump');
    }
}

Registration vs Injection #

⚠️ Main ruleГлавное правило核心规则

DO NOT register Cocos Components in the container! Components already exist on the scene — they only need injection. НЕ регистрируй Cocos Components в контейнере! Компоненты уже существуют на сцене — их нужно только инжектить. 不要将 Cocos Components 注册到容器中！组件已经存在于场景中——只需注入即可。

TypeТип类型Register?Регистрировать?注册？HowКак方式
class MyService✅builder.registerSingleton(MyService)
class MyCtrl extends Component❌@NeedInject() + @InjectProperty()
Existing instanceСуществующий instance已有实例✅builder.registerInstance(Token, instance)

✅ CorrectПравильно正确

protected configure(builder) {
  // Services
  builder.registerSingleton(GameFlowService);
  builder.registerSingleton(AudioService);
  // Instance
  builder.registerInstance(GameProvider, this.gameProvider);
}

❌ WrongНеправильно错误

protected configure(builder) {
  // DON'T register Components!
  builder.registerSingleton(GameFlowController);
  builder.registerSingleton(PlayerController);
}

Services #

Services are plain TypeScript classes, not extending Component.Сервисы — обычные TypeScript классы без наследования от Component.服务是普通的 TypeScript 类，不继承 Component。

@Injectable([ConfigService, LoggerService])
class ApiService {
    constructor(private config: ConfigService, private logger: LoggerService) {}
}
⚠️ Symbol markers requiredSymbol маркеры обязательны必须使用 Symbol 标记

Without [INITIALIZABLE] = true as const, onInitialize() will NOT be called. Markers distinguish our interfaces from Cocos methods.Без [INITIALIZABLE] = true as const метод onInitialize() НЕ будет вызван. Маркеры нужны чтобы отличить наши интерфейсы от Cocos методов.如果没有 [INITIALIZABLE] = true as const，onInitialize() 将不会被调用。标记用于区分我们的接口和 Cocos 方法。

Components #

Cocos Components receive dependencies via property injection.Cocos Components получают зависимости через property injection.Cocos Components 通过属性注入接收依赖。

@NeedInject()
@ccclass('GameController')
class GameController extends Component {
    @InjectProperty(GameService)
    private gameService!: GameService;

    start(): void {
        this.gameService.startGame();
    }
}
💡 Call orderПорядок вызовов调用顺序
  1. LifetimeScope.onLoad()
  2. injectSceneComponents()
  3. resolveAll() v1.1.0
  4. initializeAll()
  5. LifetimeScope.start() → startAll()
  6. Cocos start()

Lifecycle Interfaces #

⚠️ Services only!Только для сервисов!仅限服务！

Lifecycle interfaces work only for registered services, NOT for Cocos Components. Use start(), onDestroy() for components.Lifecycle интерфейсы работают только для зарегистрированных сервисов, НЕ для Cocos Components. Для компонентов используй start(), onDestroy().生命周期接口仅适用于注册的服务，不适用于 Cocos Components。组件请使用 start()、onDestroy()。

InterfaceMarkerMethod
IInitializableINITIALIZABLEonInitialize()
IStartableSTARTABLEonContainerStart()
ITickableTICKABLEonTick(dt)
IFixedTickableFIXED_TICKABLEonFixedTick(dt)
ILateTickableLATE_TICKABLEonLateTick(dt)
IDisposableDISPOSABLEonDispose()
 
@Injectable([])
class AudioService implements IInitializable, IDisposable {
    [INITIALIZABLE] = true as const;  // ← Required!
    [DISPOSABLE] = true as const;

    onInitialize(): void { console.log('ready'); }
    onDispose(): void { console.log('cleanup'); }
}

InjectionToken #

For registering interfaces or abstractions.Для регистрации интерфейсов или абстракций.用于注册接口或抽象类型。

interface ILogger { log(msg: string): void; }
const LOGGER = new InjectionToken<ILogger>('Logger');

@Injectable([])
class ConsoleLogger implements ILogger {
    log(msg: string): void { console.log(msg); }
}

builder.registerSingleton(LOGGER, ConsoleLogger);
⚠️ v1.1.0

Registering InjectionToken without useClass, useFactory or useValue now throws at registration, not at resolve.Регистрация InjectionToken без useClass, useFactory или useValue теперь бросает ошибку при регистрации, а не при resolve.注册 InjectionToken 时如果没有提供 useClass、useFactory 或 useValue，现在会在注册时报错，而不是在 resolve 时。

Factories #

builder.registerFactory(ApiService, (container) => {
    const config = container.resolve(ConfigService);
    const api = new ApiService(config);
    api.setBaseUrl(config.apiUrl);
    return api;
});

// Transient factory
builder.registerFactory(BulletFactory,
    (c) => new BulletFactory(c.resolve(PoolService)),
    Lifetime.Transient
);
