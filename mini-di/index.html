MiniDI

Lightweight Dependency Injection for Cocos Creator 3.x Playable Ads

MiniDI — легковесный DI-контейнер, вдохновлённый VContainer (Unity). Оптимизирован для playable ads: минимальный размер (~1.5KB gzip), без reflect-metadata.

💡 Ключевое отличие от VContainer

В VContainer можно регистрировать MonoBehaviour в контейнере. В MiniDI — Cocos Components НЕ регистрируются, а инжектятся через декораторы @NeedInject() и @InjectProperty().

Quick Start #

1. Создай сервис

@Injectable([])
class AudioService {
    play(sound: string): void { }
}

2. Создай LifetimeScope

@ccclass('GameLifetimeScope')
class GameLifetimeScope extends LifetimeScope {
    protected configure(builder: ContainerBuilder): void {
        builder.registerSingleton(AudioService);
    }
}

3. Инжекть в компонент

@NeedInject()
@ccclass('PlayerController')
class PlayerController extends Component {
    @InjectProperty(AudioService)
    private audio!: AudioService;

    start(): void {
        this.audio.play('jump');
    }
}

Registration vs Injection #

⚠️ Главное правило

НЕ регистрируй Cocos Components в контейнере! Компоненты уже существуют на сцене — их нужно только инжектить.

ТипРегистрировать?Что делать
class MyService✅ Даbuilder.registerSingleton(MyService)
class MyCtrl extends Component❌ Нет@NeedInject() + @InjectProperty()
Существующий instance✅ Даbuilder.registerInstance(Token, instance)

✅ Правильно

protected configure(builder) {
  // Сервисы — регистрируем
  builder.registerSingleton(GameFlowService);
  builder.registerSingleton(AudioService);

  // Instance — если уже создан
  builder.registerInstance(GameProvider, this.gameProvider);
}

❌ Неправильно

protected configure(builder) {
  // НЕ регистрируй Components!
  builder.registerSingleton(GameFlowController);
  builder.registerSingleton(PlayerController);
}

Как работает инъекция

// 1. LifetimeScope.onLoad() строит контейнер
// 2. injectSceneComponents() находит все @NeedInject() на сцене
// 3. resolveAll() создаёт все синглтоны
// 4. initializeAll() вызывает onInitialize()
// 5. LifetimeScope.start() → startAll()
// 6. Cocos вызывает start() на компонентах

Services #

Сервисы — обычные TypeScript классы без наследования от Component.

Без зависимостей

@Injectable([])
class ConfigService {
    readonly apiUrl = 'https://api.example.com';
}

С зависимостями

@Injectable([ConfigService, LoggerService])
class ApiService {
    constructor(
        private config: ConfigService,
        private logger: LoggerService
    ) {}
}

С lifecycle интерфейсами

@Injectable([])
class AudioService implements IInitializable, IDisposable {
    [INITIALIZABLE] = true as const;
    [DISPOSABLE] = true as const;

    onInitialize(): void {
        // Вызывается после resolveAll()
    }

    onDispose(): void {
        // Вызывается при уничтожении LifetimeScope
    }
}
⚠️ Symbol маркеры обязательны

Без [INITIALIZABLE] = true as const метод onInitialize() НЕ будет вызван. Маркеры нужны чтобы отличить наши интерфейсы от Cocos методов.

Components #

Cocos Components получают зависимости через property injection.

@NeedInject()  // Обязательно!
@ccclass('GameController')
class GameController extends Component {
    @InjectProperty(GameService)
    private gameService!: GameService;

    @InjectProperty(AudioService)
    private audio!: AudioService;

    start(): void {
        this.gameService.startGame();
    }
}
💡 Порядок вызовов
  1. LifetimeScope.onLoad() — строит контейнер
  2. injectSceneComponents() — инжектит все @NeedInject()
  3. resolveAll() — создаёт все синглтоны v1.1.0
  4. initializeAll() — вызывает onInitialize()
  5. LifetimeScope.start() → startAll()
  6. Cocos вызывает start() на всех компонентах

MethodReturns

Override

@ccclass('GameLifetimeScope')
class GameLifetimeScope extends LifetimeScope {
    @property(GameProvider)
    gameProvider: GameProvider = null!;

    protected configure(builder: ContainerBuilder): void {
        builder.registerInstance(GameProvider, this.gameProvider);
        builder.registerSingleton(GameService);
    }
}

Decorators #

@Injectable(dependencies)

Помечает класс как сервис с constructor injection.

@Injectable([ServiceA, ServiceB])
class MyService {
    constructor(private a: ServiceA, private b: ServiceB) {}
}

@NeedInject()

Помечает Cocos Component для автоматической property injection.

@NeedInject()
@ccclass('MyController')
class MyController extends Component { }

@InjectProperty(token)

Помечает property для инъекции. Безопасен при наследовании — дочерний класс не мутирует метаданные родителя.

@InjectProperty(AudioService)
private audio!: AudioService;

Lifecycle Interfaces #

⚠️ Только для сервисов!

Lifecycle интерфейсы работают только для зарегистрированных сервисов, НЕ для Cocos Components. Для компонентов используй обычные start(), onDestroy().

InterfaceMarkerMethodWhen Called
IInitializableINITIALIZABLEonInitialize()После resolveAll()
IStartableSTARTABLEonContainerStart()После initialize, перед Cocos start()
ITickableTICKABLEonTick(dt)Каждый frame (enableTickManager)
IFixedTickableFIXED_TICKABLEonFixedTick(dt)Fixed interval (1/60s)
ILateTickableLATE_TICKABLEonLateTick(dt)После всех onTick
IDisposableDISPOSABLEonDispose()При уничтожении LifetimeScope

Usage

import {
    Injectable,
    IInitializable, INITIALIZABLE,
    IDisposable, DISPOSABLE
} from './mini-di';

@Injectable([])
class AudioService implements IInitializable, IDisposable {
    [INITIALIZABLE] = true as const;  // ← Required!
    [DISPOSABLE] = true as const;     // ← Required!

    onInitialize(): void {
        console.log('Audio ready');
    }

    onDispose(): void {
        console.log('Audio cleanup');
    }
}

InjectionToken #

Для регистрации интерфейсов или абстракций.

// Объявление
interface ILogger {
    log(msg: string): void;
}

const LOGGER = new InjectionToken<ILogger>('Logger');

// Реализация
@Injectable([])
class ConsoleLogger implements ILogger {
    log(msg: string): void { console.log(msg); }
}

// Регистрация
builder.registerSingleton(LOGGER, ConsoleLogger);

// В компоненте
@InjectProperty(LOGGER)
private logger!: ILogger;
⚠️ InjectionToken без реализации — v1.1.0

Регистрация InjectionToken без useClass, useFactory или useValue теперь выбрасывает ошибку при регистрации, а не при resolve.

Factories #

Кастомная логика создания instances.

builder.registerFactory(ApiService, (container) => {
    const config = container.resolve(ConfigService);
    const logger = container.resolve(LOGGER);

    const api = new ApiService(config, logger);
    api.setBaseUrl(config.apiUrl);

    return api;
});
// Transient factory
builder.registerFactory(
    BulletFactory,
    (c) => new BulletFactory(c.resolve(PoolService)),
    Lifetime.Transient
);

Changelog #

v1.1.0 — 2026-03-11

Fixed

  • InjectProperty inheritance bug — дочерний класс мутировал массив метаданных родителя. Теперь используется hasOwnProperty + копия массива.
  • getAllSingletons force-resolve — ранее создавал ВСЕ синглтоны включая неиспользованные. disposeAll() мог создать сервис только чтобы тут же его уничтожить.
  • InjectionToken guard — register(token) с InjectionToken без реализации теперь бросает ошибку при регистрации.
  • IInitializable async — убран Promise<void> из сигнатуры, т.к. initializeAll() синхронный.

Added

  • Container.resolveAll() — eager-resolve всех синглтонов. Вызывается LifetimeScope перед lifecycle.

v1.0.0 — Initial release

  • Core DI container with @Injectable, @NeedInject, @InjectProperty
  • LifetimeScope, TickManager, Cocos Extensions
  • Lifecycle interfaces with Symbol markers
  • Logger with call tracking
